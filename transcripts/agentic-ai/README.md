## Layout

- `transcripts/full-sessions/` — one full parent-session transcript per Vimeo video, date-prefixed for chronological sort
- `transcripts/at02-per-speaker/` — AT02 time-sliced into three per-speaker files (cuts at the moderator's handoffs, accurate to ~5 s)
- `raw/` — the raw Vimeo player-config JSONs used to locate the signed `.vtt` URLs (kept for audit; VTT signatures have since expired)

Each transcript has a `.vtt` (timestamped) and `.txt` (spoken text only) pair.

## Extraction method

1. Logged into `connect.aacr26.org` through a Chrome DevTools MCP-controlled browser (Fastly CAPTCHA required manual solve).
2. Navigated to each parent session. Captured the Vimeo player config JSON at `https://player.vimeo.com/video/<id>/config?...transcript=1`.
3. Extracted `request.text_tracks[0].url` — a signed URL on `captions.vimeo.com` with the AI-generated English VTT.
4. Fetched the `.vtt` with `Referer: https://vimeo.com/`.
5. Stripped cues and timestamps to produce the `.txt` plain-text version.

Sub-talk planning pages (e.g. `Agentic AI for pathology and transcriptomic analysis`) have no independent video — the transcript lives on the parent session. To reconstruct a sub-talk on its own, time-slice the parent VTT (as done for AT02).
