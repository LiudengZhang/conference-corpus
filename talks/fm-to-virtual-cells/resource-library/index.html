    Resource library — FMs to Virtual Cells¶

    Every paper, tool, org, dataset, event, and signal we've collected on the foundation-models-to-virtual-cells story — papers we've read, models named in passing, labs to watch, money moves, competitions, perspectives. Use the filters to slice by kind, topic, status, org-type, or region.

    This is the FM-to-virtual-cells resource library — a landscape monitor, not a citation graph. It catalogues every paper, tool, lab, dataset, event, competition, blog, and signal we've collected, including items that are still just a name with a slide-trail attached. For the citable subgraph with relations and layout, see the paper map; for the methods next door see adjacent methods; for the one-line model index see the model glossary.

    How filtering works. Each card carries faceted tags (kind / status / org / region / topic). Click any chip in the filter bar to narrow. Multiple chips in the same row are OR; rows AND together. The search box matches across name, authors, venue, and the why-it-matters line.

    Source of truth. Everything here is generated from data/resource-library/fm-to-virtual-cells.yaml by scripts/resource_library_build.py. Add an item to the YAML, rerun the build, and it lands on the page with its tags wired up. The YAML is also the export point for papermap: items with kind: paper and papermap_category: set are export-eligible.
